## 摘要
大型语言模型（LLMs）存在幻觉和上下文遗忘问题。先前的研究表明，注意力漂移是这些问题的主要原因之一，即 LLM 的关注点从初始输入上下文转移到新生成的标记上。为了解决这一问题，我们利用 LLM 的一个相关的内在特性——注意力吸收（attention sink），即模型倾向于持续对序列的首个标记（即 <BOS>）分配高注意力。具体而言，我们提出一种改进的上下文锚定方法 SinkTrack，将 <BOS> 视为信息锚点，并将关键的上下文特征（例如来自输入图像或指令的特征）注入其表示中。这样，LLM 在整个生成过程中都能保持与初始输入上下文的锚定。SinkTrack 无需训练，可直接使用，且推理开销可忽略不计。实验结果表明，SinkTrack 能够减轻文本任务（例如在 Llama3.1-8B-Instruct 上的 SQuAD2.0 提升 +21.6%）和多模态任务（例如在 Qwen2.5-VL-7B-Instruct 上的 M3CoT 提升 +22.8%）中的幻觉和上下文遗忘问题。其在不同架构和规模上的稳定增益凸显了方法的鲁棒性与泛化能力。我们还从信息传递的角度分析了其底层工作机制。源代码可在 https://github.com/67L1/SinkTrack 获取。

## Abstract
Large language models (LLMs) suffer from hallucination and context forgetting. Prior studies suggest that attention drift is a primary cause of these problems, where LLMs' focus shifts towards newly generated tokens and away from the initial input context. To counteract this, we make use of a related, intrinsic characteristic of LLMs: attention sink -- the tendency to consistently allocate high attention to the very first token (i.e., <BOS>) of a sequence. Concretely, we propose an advanced context anchoring method, SinkTrack, which treats <BOS> as an information anchor and injects key contextual features (such as those derived from the input image or instruction) into its representation. As such, LLM remains anchored to the initial input context throughout the entire generation process. SinkTrack is training-free, plug-and-play, and introduces negligible inference overhead. Experiments demonstrate that SinkTrack mitigates hallucination and context forgetting across both textual (e.g., +21.6% on SQuAD2.0 with Llama3.1-8B-Instruct) and multi-modal (e.g., +22.8% on M3CoT with Qwen2.5-VL-7B-Instruct) tasks. Its consistent gains across different architectures and scales underscore the robustness and generalizability. We also analyze its underlying working mechanism from the perspective of information delivery. Our source code is available at https://github.com/67L1/SinkTrack.

## 一、研究问题与背景

- **核心问题**：大型语言模型（LLMs）在生成过程中普遍面临两类关键缺陷：

- **诱因分析**：论文指出这些问题主要源于 **注意力漂移（Attention Drift）**。即在自回归生成中，模型的注意力逐步集中于最新生成的 token，而初始输入的 token 注意力急剧下降。
- **重要观察**：作者发现一种逆向特性——**注意力吸收（Attention Sink）**。首个标记 `<BOS>` 即使语义稀薄，却保持高且稳定的注意力权重。
- **研究动机**：若能利用这种稳定性，将 `<BOS>` 转化为“信息锚点”，可在整个生成过程中保持模型对源上下文的关注，从而缓解幻觉与遗忘。

## 二、方法论：SinkTrack 的核心思想与技术机制

### 1. 核心概念
- **总体思想**：将 `<BOS>` 从被动的注意力吸收点转化为主动的信息锚点，通过对其内部表示进行上下文信息注入，使其在后续层持续携带输入上下文。
- **性质**：完全 **训练无关（training-free）**、**即插即用（plug-and-play）**、几乎不增加推理开销。

### 2. 方法演进
作者经历三个阶段的探索：

1. **硬注入（Hard Injection）**
   - 直接替换 `<BOS>` 在 KV cache 中对应的 Value 向量为外部信息向量 \(f_{info}\)。
   - 结果：模型崩溃，生成失效 → 说明此操作破坏了模型的内部计算流。

2. **软注入（Soft Injection）**
   - 以公式融合方式引入上下文信息：
     \[
     \bar{H}^{(l)}_0 = \alpha H^{(l)}_0 + (1-\alpha) f_{info}
     \]
     其中 \(\alpha\) 为注入强度，\(H^{(l)}_0\) 为 `<BOS>` 的原始隐状态。
   - 效果：显著提升性能，但需手动调节 α，且可能引入噪声。

3. **最终方案：SinkTrack（Dual-Track Cross-Attention）**
   - **双轨设计**：
     - **Track 1（注入轨）**：在注入层对 `<BOS>` 使用跨注意力机制：
       \[
       \bar{H}^{(l)}_0 = \text{MHA}(Q=H^{(l)}_0, K=f_{info}, V=f_{info})
       \]
       使 `<BOS>` 动态检索上下文中的关键特征。
     - **Track 2（保留轨）**：其余 token 保持标准的因果自注意力，不受干扰。
   - **结果拼接**：两条轨的输出拼接后经归一化与前馈层流通，保持原架构的完整性。
   - **改进细节**：去除信息的 mean-pooling 步骤（直接使用原始序列特征）以防信息丢失。
   - **注入位置策略**：每 5 层执行一次注入效果最佳，避免过频扰动。
